1. 데이터 통신과 컴퓨터 네트워크
  1단원 네트워크의 기초
    48p 연습문제
	
	1. 다음 용어를 설명하시오.
		시스템: 호스트. 네트워크를 사용하는 시스템. 컴퓨터.
						-> 체제. 내부 규칙에 따라 능동적으로 동작하는 대상
		인터페이스: 네트워크를 접하는 부분?
						-> 시스템과 시스템을 연결하기 위한 표준화되 접근 방법이다.
							논리규격, 물리 규격의 표준화가 필요하다.
		전송매체: 케이블. 전송을 위한 물리적 연결
						-> 물리적 전송수단
		프로토콜: 계층 간 전송을 위한 규약
						-> 동등한 위치에 있는 시스템 사이의 규칙이라는 것이 특징
		네트워크: 
						-> 여러 시스템이 프로토콜을 통헤 데이터를 주고받을 때
							이들을 하나의 단위로 통칭하여 네트워크라고 부른다.
		인터넷: 
						-> 전 세계의 네트워크. 통합 네트워크.
		
	2. 다음 용어의 차이점을 설명하시오.
		노드: 데이터를 주고받을 수 있는 시스템의 통칭
		호스트: 컴퓨팅 기능이 있는 시스템
		클라이언트: 요청한 시스템
		서버: 요청받은(제공하는) 시스템
		
	3. OSI 7계층 모델의 각 계층 기능을 설명하시오.
		물리 계층: 전송매체와 연결하기 위한 인터페이스 규칙과 전송 캐체의 특성을 다룬다.
		데이터링크 계층: 물리적 전송 오류를 감지하는 기능 제공. 
						(오류의 종류: 데이터 분실/데이터 변형)
		네트워크 계층: 수신 호스트까지 올바른 경로를 선택하도록 지원한다.
						IP 프로토콜, IP와 관련 있다.
		전송 계층 :송수신 프로세스 사이의 연결 기능을 제공한다.
		세션 계층: 전송계층과 유사하지만, 더 상위의 논리적 연결이다.
		표현 계층: 압축, 암호화
		응용 계층: 네트워크 응용환경을 지원한다.
		
	
	4. 계층 구조 모델에서 프로토콜과 인터페이스의 차이를 설명하시오.

	7. 호스트를 구분하는 구분자의 특징을 나열하시오.
		주소: 시스템을 구분할 수 있는 고유 구분자이다. 32비트로 이루어져 있다(IPv4)
		이름: 사용자가 이해하기 쉽도록 만든 구분자이다.
	
	8. 32비트의 IP 주소를 4자리의 10진수로 축약하여 표현하는 방법을 설명하시오.
		각각 8비트씩 4개로 나눠 10진수로 표현한다.
		이것이 우리가 보통 알고 있는 IPv4의 IP이다.
	
	9. 호스트 이름의 필요성과 구조를 설명하시오.
		IP는 시스템이 이해하기 쉬운 숫자형태이다. 
		이는 인간이 이해하기 어렵기 때문에 호스트 이름이 필요한 것이다.
		구조는 주로 www.naver.com과 같다.

		DNS는 Domain Name System의 약자로 호스트 이름을 IP 주소로 변환해 주는 
		시스템이다. 도메인과 IP의 매핑을 각각의 시스템에 저장하는 것은 무리이므로

		MAC: 랜카드 고유 번호. 데이터링크 계층에서 사용한다.
