1. 데이터 통신과 컴퓨터 네트워크
  - chapter 2 네트워크 모델 74p 연습문제
  
  1. 모듈화된 계층 구조 프로토콜의 장점을 설명하시오.
  
  	- 전체 시스템을 이해/설계/구현하기 쉽다.
  	- 모듈이 독립적이기 때문에, 테스트/유지보수에 강점을 지닌다.
  
  
  2. 계층 구조의 네트워크 프로토콜을 설계할 때 고려할 사항을 나열하시오.
  
  	- 주소표현
  		브로드캐스팅(Broadcasting) 표기방법: 네트워크의 모든 호스트에게 데이터를 전송
  		멀티캐스팅(Multicasting) 표기방법: 네트워크의 특정 호스트를 그룹으로 묶어 데이터를 전송
  		
  	- 오류제어: 보통 재전송(Retransmission)으로 해결한다.
  		데이터 분실
  		데이터 변형
  		
  	- 흐름제어: 수신 호스트의 버퍼 처리속도와 송신 호스트의 데이터 전송속도의 
  				gap이 생길 수 있기에 필요

  	- 데이터 전달방식
  		단방향(Simplex)
  		전이중(Full Duplex): 양방향 통신
  		반이중(Half Duplex): 특정 시점에 한방향
  
  
  3. 임의의 송신 호스트가 수신 호스트에 데이터를 전송했을 때 발생할 수 있는 오류는 무엇이 있는가?
  
  	데이터 분실, 데이터 변형
  
  4. 흐름 제어 기능을 설명하시오.
  
  	수신 호스트는 송신 호스트가 보낸 데이터를 버퍼에 보관했다가 처리한다. 
  	그런데 만약에 수신 호스트의 버퍼 처리 속도가 송신 데이터 속도보다 느려서
  	버퍼 공간을 확보하지 못한다면 데이터를 분실하는 오류가 생기게 된다.
  	이를 제어하기 위해 흐름 제어가 필요하다.
  	흐름 제어는 보통 수신 호스트가 송신 호스트에게 순차적으로 데이터 전송 허가를
  	보냄으로서 이루어진다.
  	
  
  5. 서비스 프리미티브에서 Request, Indication, Response, Confirm의 기능을 설명하시오.
  
  	Request: 서비스 요청
  	Indication: 서비스 요청 도착
  	Response: 서비스 응답
  	Confirm: 서비스 응답 도착
  
  
  6. OSI 7계층 모델에서 각 계층을 설명하시오.
  
  	응용계층: 응용환경(응용프로그램)에 필요한 기능을 다룬다.
  	표현계층: 데이터의 의미, 표현방법을 담당한다. 암호화, 압축 등과 관계있다.
  	세션계층: 세션기능(예.로그인 등)을 담당한다.
  	전송계층: End-to-End. ㅡㅍ로세스와 프로세스의 연결을 담당한다.
  	네트워크 계층: 라우팅 문제를 담당한다. 전달경로를 결정한다.
  	데이터링크 계층: 물리적 전송오류를 담당/해결한다. 패킷의 신뢰성을 보장한다.
  	물리계층: 물리적 인터페이스를 담당한다.
  	
  	
  7. TCP/IP 모델의 구현환경을 설명하시오.
  
  	시스템공간
  		TCP는 커널에 구현된다. 사용자는 소켓 인터페이스를 통해 이를 호출하여 사용할 수 있다.
  		IP를 통해 네트워크 계층을 구현한다. 그 아래의 데이터링크와 물리계층은 LAN 카드와 드라이버 루틴을
  		통해 동작된다.
  	사용자공간
  		소켓 인터페이스를 통해 전송계층인 TCP를 호출한다. 소켓에는 포트번호로 고유번호를
  		부여해야 한다. 
  
  
  8. ARP, RARP의 필요성을 설명하시오.
  	목적지의  MAC주소를 모를 경우 IP를 통해 MAC 주소를 알아내는 기능이 필요하다.
  	반대의 경우 MAC주소를 IP주소로 변환하는 기능이 필요하다.
